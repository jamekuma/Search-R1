# Search-R1 学习大纲

本文档是整套讲义的总导航。我会持续以它为准往下写，避免后续章节偏题或重复。

为减少术语漂移，后文默认采用下面这套叫法：

- **检索服务**：对外暴露 `/retrieve` 接口的 HTTP 服务；
- **检索器（retriever）**：检索服务内部真正执行召回的模块，例如 BM25 或 Dense Retriever；
- **搜索工具**：站在 agent 视角时，对“可被模型调用的外部检索能力”的统称。

## 1. 这套讲义适合谁

这套讲义按下面的读者画像来写：

- 你熟悉机器学习、深度学习、Python、PyTorch；
- 你对 LLM 训练流程只有模糊概念；
- 你知道强化学习的基本术语，但对策略梯度、PPO、GRPO 还不熟；
- 你的目标不是“只会跑脚本”，而是**真正理解 Search-R1 为什么这样设计、代码如何把设计落地**。

因此，这套讲义会遵循一个原则：

> 先讲清楚“它在解决什么问题、整体流程是什么”，再下钻到关键代码。

不会一开始就把你扔进大段分布式训练代码里。

---

## 2. 建议学习顺序

建议按下面的顺序学习：

1. 先建立全局图景：Search-R1 到底是什么；
2. 再看最小闭环：模型如何一边生成、一边搜索；
3. 然后先把代码跑起来：知道最小可运行路径、依赖环境和适合下断点的位置；
4. 再看训练主流程：RL 是怎么把这种能力训出来的；
5. 最后再看 reward、PPO/GRPO、数据格式和可扩展点。

如果你现在直接去看 `verl/trainer/ppo/ray_trainer.py`，大概率会被大量工程细节淹没。

## 3. 全部讲义规划

### 第一部分：先把问题想明白

#### 第 01 章：先修背景——从 RAG、Agent 到 PPO/GRPO
目标：
- 理解 Search-R1 与普通 RAG 的区别；
- 建立“推理 + 搜索 + 强化学习”的统一视角；
- 补足你后面读代码一定会遇到的 RL 背景词汇。

#### 第 02 章：代码库全景图
目标：
- 看懂仓库顶层目录；
- 知道 `search_r1/` 和 `verl/` 分别负责什么；
- 知道新手应该优先看哪些文件，哪些文件可以后看。

#### 第 03 章：从 `infer.py` 看最小工作闭环
目标：
- 用最少的代码理解 Search-R1 的核心运行方式；
- 理解 `<search>...</search>`、`<information>...</information>`、`<answer>...</answer>` 这些标签在系统中的角色；
- 理解“模型生成 -> 搜索 -> 回填上下文 -> 再生成”的循环。

### 第二部分：先把代码跑起来，再进入训练

#### 第 04 章：如何把 Search-R1 跑起来，并用 `pdb` 调试
目标：
- 搭起最小可运行环境；
- 先跑通推理与检索服务，再决定是否进入训练；
- 给出适合新手理解执行流的 `pdb` 调试入口、推荐断点和最小调试命令。

#### 第 05 章：从 `train_ppo.sh` 到 `main_ppo.py`
目标：
- 看懂训练是怎么启动的；
- 理解 Hydra 配置、训练参数和入口脚本的关系；
- 弄清 PPO 与 GRPO 在入口层面的区别。

#### 第 06 章：`RayPPOTrainer.fit()` 是整套训练的主战场
目标：
- 看懂 batch 是怎么被 rollout、打分、算 advantage、更新 actor/critic 的；
- 明确 Search-R1 相比普通 PPO 的关键增量到底在哪里。

#### 第 07 章：`generation.py`——Search-R1 最核心的 agent 循环
目标：
- 逐步看懂 `LLMGenerationManager.run_llm_loop()`；
- 理解多轮搜索、非法动作处理、observation 回填、info mask 等机制；
- 理解为什么这个文件是“Search-R1 的灵魂”。

### 第三部分：把 RL 细节讲透

#### 第 08 章：奖励函数、数据格式与 `qa_em.py`
目标：
- 看懂训练样本长什么样；
- 看懂 rule-based reward 如何工作；
- 理解为什么 Search-R1 不一定依赖复杂 reward model。

#### 第 09 章：PPO、GAE 与 GRPO 在代码里的实现
目标：
- 结合 `core_algos.py` 理解 advantage、return、policy loss、value loss；
- 理解 GAE 与 GRPO 的差别；
- 弄清为什么 GRPO 不需要 critic。

### 第四部分：搜索系统与扩展实践

#### 第 10 章：检索服务、语料与索引
目标：
- 理解 `retrieval_server.py`、`retrieval.py` 的工作方式；
- 看懂 BM25、Dense Retriever、FAISS 索引在这个项目里的位置；
- 知道如何替换成你自己的语料和搜索引擎。

#### 第 11 章：如何改造和复用 Search-R1
目标：
- 如果你想迁移到自己的任务，需要改哪些地方；
- 哪些模块是“任务相关”的，哪些是“框架通用”的；
- 如何避免常见误区。

## 6. 读这套讲义时你最应该关注什么

对你来说，我建议重点盯住下面三个问题：

1. **Search-R1 到底比普通 RAG 多了什么？**
2. **“搜索”在代码里到底被当成什么对象？是 prompt 技巧、工具调用，还是 RL 中的 action？**
3. **PPO / GRPO 到底是在优化“回答能力”，还是在优化“什么时候搜、怎么搜、搜完如何继续推理”的策略？**

如果这三个问题你能答清楚，这个项目你就算真正入门了。
